# Once Props

Once props are cached on the client and not resent on subsequent visits.

## When to Use

- Configuration data
- Feature flags
- User permissions
- Static reference data

## Definition

```typescript
import { createInertia, prop } from 'inertia-server';

const { definePage } = createInertia({ version: '1.0.0', render: renderToHtml });

const appPage = definePage({
  component: 'App',
  props: {
    // Refreshed on every visit
    user: prop<User>(),
    
    // Cached on first load
    config: prop<Config>().once(),
    features: prop<string[]>().once(),
  },
});
```

## Usage

```typescript
app.get('/', ({ inertia }) => {
  return inertia.render(appPage({
    user: getCurrentUser(),
    config: {
      theme: 'dark',
      locale: 'en-US',
    },
    features: ['feature-a', 'feature-b'],
  }));
});
```

## How It Works

1. First visit: All props are sent
2. Subsequent visits: Once props are skipped
3. Full page reload: Once props are sent again

## Expiration

Set an expiration time for once props:

```typescript
const settingsPage = definePage({
  component: 'Settings',
  props: {
    // Expires after 5 minutes
    plans: prop<Plan[]>().once({ expiresAt: Date.now() + 5 * 60 * 1000 }),
  },
});
```

## Forcing Refresh

To force refresh once props, use a full page reload or clear the client cache:

```tsx
// Force full page reload
window.location.reload();

// Or use Inertia's replace with forced reload
router.reload({ only: ['config'] });
```

## Best Practices

1. **Use for static data** - Only use for data that rarely changes
2. **Set expiration** - For time-sensitive data, use expiration
3. **Consider user actions** - Users logging in/out should refresh
