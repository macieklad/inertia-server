# Deferred Props

Deferred props are loaded after the initial page render, allowing for faster initial page loads.

## When to Use

- Heavy database queries
- External API calls
- Analytics data
- Non-critical content

## Definition

```typescript
import { createInertia, prop } from 'inertia-server';

const { definePage } = createInertia({ version: '1.0.0', render: renderToHtml });

const dashboardPage = definePage({
  component: 'Dashboard',
  props: {
    // Loads immediately
    title: prop<string>(),
    
    // Loads after initial render
    stats: prop<Stats>().deferred(),
    analytics: prop<Analytics>().deferred(),
  },
});
```

## Usage

Deferred props accept async functions:

```typescript
app.get('/dashboard', ({ inertia }) => {
  return inertia.render(dashboardPage({
    title: 'Dashboard',
    stats: async () => {
      return await fetchStats(); // Takes 2 seconds
    },
    analytics: async () => {
      return await fetchAnalytics(); // Takes 3 seconds
    },
  }));
});
```

## Client-Side Handling

Use the `Deferred` component from `@inertiajs/react`:

```tsx
import { Deferred } from '@inertiajs/react';

export default function Dashboard({ title, stats, analytics }) {
  return (
    <div>
      <h1>{title}</h1>
      
      <Deferred data="stats" fallback={<StatsLoading />}>
        <Stats data={stats} />
      </Deferred>
      
      <Deferred data="analytics" fallback={<AnalyticsLoading />}>
        <Analytics data={analytics} />
      </Deferred>
    </div>
  );
}
```

## Deferred Groups

Group related deferred props to load together:

```typescript
const dashboardPage = definePage({
  component: 'Dashboard',
  props: {
    chart1: prop<ChartData>().deferred('charts'),
    chart2: prop<ChartData>().deferred('charts'),
    table: prop<TableData>().deferred('table'),
  },
});
```

## Best Practices

1. **Show loading states** - Always provide meaningful fallback UI
2. **Prioritize content** - Load above-the-fold content immediately
3. **Group related data** - Use deferred groups for related data
4. **Handle errors** - Implement error boundaries for failed loads
