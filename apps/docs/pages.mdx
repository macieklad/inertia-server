# Pages

Pages in Inertia Server are defined using the `definePage()` function from your Inertia instance, which provides full type safety for your page props.

## Basic Page Definition

```typescript
import { createInertia, prop } from 'inertia-server';

const { definePage } = createInertia({
  version: '1.0.0',
  render: (page) => renderToHtml(page),
});

const homePage = definePage({
  component: 'Home',
  props: {
    title: prop<string>(),
    description: prop<string>(),
  },
});
```

The `component` property is the component name that will be resolved on the client side.

## Using Pages in Routes

```typescript
app.get('/', ({ inertia }) => {
  return inertia.render(homePage({
    title: 'Welcome',
    description: 'Hello, World!',
  }));
});
```

## Page with Complex Props

```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

const profilePage = definePage({
  component: 'Profile',
  props: {
    user: prop<User>(),
    posts: prop<Post[]>(),
    isOwner: prop<boolean>(),
  },
});
```

## Nested Component Names

Use slashes for nested page components:

```typescript
const usersIndexPage = definePage({
  component: 'Users/Index',
  props: {
    users: prop<User[]>(),
  },
});

const usersEditPage = definePage({
  component: 'Users/Edit',
  props: {
    user: prop<User>(),
  },
});
```

## Client-Side Resolution

```tsx
// src/ui/main.tsx
createInertiaApp({
  resolve: (name) => {
    const pages = import.meta.glob('./pages/**/*.tsx', { eager: true });
    return pages[`./pages/${name}.tsx`];
  },
  // ...
});
```

This will resolve:
- `'Home'` → `./pages/Home.tsx`
- `'Users/Index'` → `./pages/Users/Index.tsx`
- `'Users/Edit'` → `./pages/Users/Edit.tsx`
